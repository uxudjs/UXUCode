# Commit Message

Inspect the actual diff, repository convention, and validation evidence. Propose a terse imperative subject that describes the user-visible or architectural outcome, plus a body only when rationale, breaking behavior, migration, or verification deserves preservation.

Do not stage, commit, push, or claim validation unless the user authorized those actions and the evidence exists. Mark breaking changes explicitly.
