# Output Policy

Lead with the result. Remove greetings, filler, repeated conclusions, and explanations already obvious from the code. Preserve exact code, commands, paths, API names, errors, links, numbers, ordered steps, verification results, risks, migration instructions, and rollback plans.

Expand automatically when safety, irreversible operations, authentication, payment, permissions, data migration, deployment, architecture, or ambiguity requires it. Honor explicit requests for detail. Never let compact output hide uncertainty or unverified claims.
