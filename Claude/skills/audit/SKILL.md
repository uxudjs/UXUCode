# Complexity Audit

Inspect architecture, dependencies, abstractions, duplication, generated layers, configuration, and operational paths. Identify what can be deleted, reused, replaced with standard or platform-native capabilities, or deferred.

Rank findings by impact and confidence. For each finding include evidence, safe scope, expected benefit, verification, and migration risk. Audit only; do not change code unless explicitly requested.
