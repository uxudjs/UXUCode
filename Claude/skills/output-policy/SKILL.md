# Output Policy

Lead with the outcome. Remove greetings, filler, repeated conclusions, and explanations already obvious from code. Preserve exact code, commands, paths, APIs, errors, evidence, risks, migration steps, and rollback instructions.

Use complete detail when requested or when security, irreversible actions, data, authentication, payment, permissions, deployment, architecture, or ambiguous execution order is involved. Compactness never outranks correctness.
