# Debt

Find `uxucode-debt:` markers and other clearly documented compromises. For each item report:

- exact location;
- reason it exists;
- current bound or limit;
- trigger that requires an upgrade;
- owner if recorded;
- verification or monitoring needed.

Reject vague debt markers. A valid marker must describe the constraint and a measurable escalation condition. Do not create speculative debt items.
