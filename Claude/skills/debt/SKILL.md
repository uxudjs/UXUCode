# Technical Debt

Find `uxucode-debt:` markers and other clearly documented bounded shortcuts. For each item record location, current limit, risk, owner if known, verification coverage, and the measurable condition that requires escalation.

Do not treat every TODO as technical debt. Flag unbounded or unjustified shortcuts as findings. Do not create compatibility layers merely to avoid a breaking decision.
